# Sprint 09 – MongoDB  
Data Exploration and Querying in a Cinematic Database

This repository contains exercises focused on applying MongoDB to explore, query, and analyze a dataset designed for a cinema‑related application. The sprint evaluates the ability to work with real‑world NoSQL data structures, perform analytical queries, and navigate collections using **MongoDB** 

---

## Overview of the Database

The database simulates the backend of a movie‑streaming or cinema‑management platform. It includes the following collections:

- **users** — Stores user information such as name, email, and encrypted passwords.  
- **theaters** — Contains cinema details, including addresses and geographic coordinates.  
- **sessions** — Tracks user authentication sessions and JWT tokens.  
- **movies** — Holds movie metadata: plot, genres, runtime, cast, comments, release year, directors, awards, and more.  
- **comments** — Includes user‑generated comments linked to specific movies, with author, text, and timestamp.

These collections provide a realistic environment for practicing document‑based data analysis.

---

## Exercises
The exercises cover filtering, logical operators, projections, sorting, and document counting—core skills for working with MongoDB.

---

## Technology Stack

### Database & Tools
- **MongoDB Community Edition** — NoSQL document‑oriented database.  
- **MongoDB Shell (mongosh)** — Command‑line interface for executing queries and scripts.  

### Environment
- Local MongoDB instance running on `localhost:27017`.  
- JSON/BSON datasets provided for collection import.

---

## Sprint Objective

The primary goal of this sprint is to demonstrate proficiency in:

- Understanding and navigating a document‑oriented database.  
- Executing analytical queries using MongoDB.  
- Applying logical operators, projections, and sorting.  
- Interpreting results through both CLI and GUI tools.  
- Working with interconnected collections in a realistic dataset.
